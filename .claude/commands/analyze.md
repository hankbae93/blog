# /analyze - 인사이트 추출 커맨드

수집된 데이터로부터 **진짜 기회가 있는** MVP 아이디어를 도출합니다.

## 트랙 옵션

| 커맨드 | 설명 |
|--------|------|
| `/analyze` | 투 트랙 실행 (essence + profit) |
| `/analyze:essence` | 🧠 본질 추구형만 |
| `/analyze:profit` | 💰 수익화 특화만 |

---

## 핵심 원칙

### ❌ 피해야 할 아이디어
1. **Kill Zone 아이디어**: 대기업(Google, Microsoft, OpenAI 등)이 이미 하고 있거나, 쉽게 따라할 수 있는 영역
2. **Me-too 아이디어**: 기존 인기 서비스의 단순 클론 (예: "X의 Y 버전")
3. **Feature 아이디어**: 기존 제품의 한 기능일 뿐인 것 (Notion이 내일 추가할 수 있는 것)
4. **이미 포화된 시장**: AI 코드 어시스턴트, 범용 챗봇, AI 이미지 생성 등

### ✅ 찾아야 할 아이디어
1. **니치 마켓**: 대기업이 관심 없는 작은 시장, 특정 직군/산업 타겟
2. **로컬 기회**: 한국/아시아 시장 특화, 언어/문화 장벽이 있는 영역
3. **통합 기회**: 여러 도구를 연결하는 글루(glue) 서비스
4. **타이밍 기회**: 새 기술/플랫폼 등장으로 방금 가능해진 것
5. **워크플로우 기회**: 특정 직군의 반복 작업 자동화

## 분석 프레임워크

### Step 1: 트렌드 클러스터링
수집된 데이터에서 패턴을 찾되, **원천 기술**과 **응용 영역**을 구분:

| 원천 기술 | 응용 가능 영역 |
|-----------|----------------|
| RAG/Memory | 특정 도메인 지식 관리 |
| Voice AI | 특정 언어/방언/상황 |
| Multi-agent | 특정 워크플로우 자동화 |
| TUI/CLI | 특정 직군 도구 |

### Step 2: Kill Zone 체크 (필수!)

각 아이디어에 대해 반드시 확인:

```markdown
**Kill Zone 체크:**
- [ ] Google이 이미 하고 있나? (Search, Workspace, Cloud)
- [ ] Microsoft가 이미 하고 있나? (Copilot, Office, Azure)
- [ ] OpenAI/Anthropic이 이미 하고 있나?
- [ ] 잘 알려진 스타트업이 $10M+ 펀딩받고 하고 있나?
- [ ] 기존 대형 서비스의 한 기능으로 쉽게 추가 가능한가?

→ 2개 이상 체크되면 **아이디어 폐기** 또는 **니치 피벗 필수**
```

### Step 3: 차별화 전략 명시

아이디어가 살아남으려면 다음 중 최소 하나의 명확한 차별화가 필요:

| 전략 | 설명 | 예시 |
|------|------|------|
| **니치 타겟** | 대기업이 무시하는 작은 시장 | "프리랜서 번역가 전용", "수의사 전용" |
| **로컬라이징** | 특정 국가/언어 특화 | "한국 법률 문서", "일본 이커머스" |
| **워크플로우 통합** | 여러 도구 연결 | "Slack + Linear + GitHub 통합" |
| **가격 파괴** | 90% 저렴하게 | "GPT-4 품질을 1/10 가격에" |
| **오프라인/프라이버시** | 로컬 실행, 데이터 보호 | "회사 밖으로 데이터 안 나감" |
| **하드웨어 결합** | 소프트웨어만으론 안 되는 것 | IoT, 웨어러블 연동 |

### Step 4: "왜 지금?" 타이밍 분석

좋은 아이디어는 **지금 이 시점에 가능해진 이유**가 있어야 함:
- 새 API/플랫폼 출시 (예: Claude MCP, Apple Vision Pro)
- 규제 변화 (예: 새 법률, 인증 요구)
- 비용 급감 (예: LLM API 가격 하락)
- 행동 변화 (예: 원격근무 정착)

### Step 5: 창의적 교차 조합

**금지:** 단일 소스에서 본 것을 그대로 따라하기
**필수:** 최소 2개 소스의 트렌드를 조합하여 새로운 것 만들기

```
좋은 예:
- HN(자세교정앱) + IndieHackers(B2B SaaS) → "원격근무 팀용 웰니스 대시보드"
- GitHub(RAG) + GeekNews(한국) → "한국 법률/세무 문서 특화 RAG"

나쁜 예:
- PH에서 본 앱을 그대로 클론
- GitHub 트렌딩 프로젝트에 UI만 씌우기
```

### Step 6: 페르소나 기반 평가 (서브에이전트 실행)

**중요:** 이 단계에서는 반드시 Task 도구로 서브에이전트를 실행하여 페르소나 기반 평가를 수행합니다.

## 🧠 ESSENCE TRACK (본질 추구형)

**페르소나 파일:** `generated/my-persona/essence-persona.md`

### 평가 프레임워크

| # | 질문 | 평가 기준 |
|---|------|-----------|
| 1 | **전제 파괴** | 이 제품은 어떤 전제를 깨거나 드러내는가? |
| 2 | **방치된 이유** | 이 문제는 왜 지금까지 방치되었는가? |
| 3 | **비합리적 행동** | 이 제품이 없을 때 사용자는 어떤 비합리한 행동을 하고 있는가? |
| 4 | **기능-가설 연결** | 기능 하나하나가 문제 가설과 직접 연결되는가? |
| 5 | **진정성** | 이걸 만드는 사람이 문제를 진짜 겪어본 사람처럼 보이는가? |

### 서브에이전트 호출
```
Task 도구:
- subagent_type: "general-purpose"
- prompt:
  "generated/my-persona/essence-persona.md 파일을 읽고, 아래 아이디어들을 평가해주세요.
   [아이디어 목록]
   각 아이디어에 대해 5가지 질문으로 ✅/⚠️/❌ 판정 후 종합 판정 작성"
```

### 평가 결과 형식

```markdown
### 🧠 본질 평가: [아이디어명]

| 질문 | 판정 | 근거 |
|------|------|------|
| 전제 파괴 | ✅/⚠️/❌ | [구체적 설명] |
| 방치된 이유 | ✅/⚠️/❌ | [구체적 설명] |
| 비합리적 행동 | ✅/⚠️/❌ | [구체적 설명] |
| 기능-가설 연결 | ✅/⚠️/❌ | [구체적 설명] |
| 진정성 | ✅/⚠️/❌ | [구체적 설명] |

**종합 판정:** ✅ 추천 / ⚠️ 보완 필요 / ❌ 폐기
**핵심 피드백:** [날카로운 질문이나 반박]
```

→ **하나라도 ❌이면:** 보류 또는 폐기 권고
→ **모두 ✅이면:** PRD 생성 우선순위 부여

## 💰 PROFIT TRACK (수익화 특화)

**페르소나 파일:** `generated/my-persona/profit-persona.md`

### 평가 프레임워크

| # | 질문 | 평가 기준 |
|---|------|-----------|
| 1 | **지불자** | 누가 돈을 내는가? 지불자가 명확한가? |
| 2 | **가격** | 얼마를 내는가? 구체적 가격이 있는가? |
| 3 | **지불 동기** | 왜 지금 돈을 내는가? 지불 동기가 강한가? |
| 4 | **기존 지출** | 이미 돈을 쓰고 있는가? 기존 지출을 대체하는가? |
| 5 | **첫 수익 속도** | 첫 수익까지 얼마나 걸리는가? 30일 내 가능한가? |
| 6 | **반복 매출** | 반복 매출인가? 구독/리텐션 구조인가? |
| 7 | **단위 경제** | 단위 경제가 맞는가? CAC < LTV인가? |

### 서브에이전트 호출
```
Task 도구:
- subagent_type: "general-purpose"
- prompt:
  "generated/my-persona/profit-persona.md 파일을 읽고, 아래 아이디어들을 수익화 관점에서 평가해주세요.
   [아이디어 목록]
   각 아이디어에 대해 7가지 질문으로 ✅/⚠️/❌ 판정 후 종합 판정 작성.
   추천 가격대도 제안해주세요."
```

### 평가 결과 형식

```markdown
### 💰 수익 평가: [아이디어명]

| 질문 | 판정 | 근거 |
|------|------|------|
| 지불자 | ✅/⚠️/❌ | [누가 지불하는가] |
| 가격 | ✅/⚠️/❌ | [제안 가격: $XX/월] |
| 지불 동기 | ✅/⚠️/❌ | [왜 돈을 내는가] |
| 기존 지출 | ✅/⚠️/❌ | [대체하는 기존 비용] |
| 첫 수익 속도 | ✅/⚠️/❌ | [예상 기간] |
| 반복 매출 | ✅/⚠️/❌ | [구독 모델 적합성] |
| 단위 경제 | ✅/⚠️/❌ | [CAC vs LTV 추정] |

**종합 판정:** ✅ 추천 / ⚠️ 보완 필요 / ❌ 폐기
**예상 MRR:** $X,XXX (첫 6개월)
**추천 가격:** $XX/월
**핵심 피드백:** [수익화 개선 제안]
```

→ **3개 이상 ❌이면:** 폐기 또는 피벗
→ **5개 이상 ✅이면:** PRD 생성 우선순위 부여
