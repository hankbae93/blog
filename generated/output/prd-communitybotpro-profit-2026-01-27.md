## 1. 개요

### 1.1 문제 정의

- **누가** 돈을 내는가?
  - 유료 커뮤니티 운영자 (코스 크리에이터, 멤버십 커뮤니티)
  - SaaS 기업의 커뮤니티 매니저
  - 인플루언서/콘텐츠 크리에이터의 팬 커뮤니티 운영자

- **무엇이** 문제인가?
  - **반복 업무 과부하**: 새 멤버 환영, FAQ 응답에 하루 2-3시간 소모
  - **미응답 질문 방치**: 답변 없는 질문이 쌓여 멤버 불만 증가
  - **이탈 조기 감지 실패**: 비활성 멤버를 놓쳐 갱신율 하락
  - **톤앤매너 불일치**: 여러 모더레이터 간 응대 품질 편차
  - **24/7 대응 불가**: 글로벌 커뮤니티의 시간대 이슈

- **왜** 돈을 내는가?
  - VA(가상비서) 인건비 $500-2,000/월 대체
  - 커뮤니티 리텐션 = 구독 수익 직결 (이탈률 5% 감소 = 연 매출 20% 증가)
  - 관리 시간 절감 -> 콘텐츠 제작/비즈니스 확장에 투자

### 1.2 솔루션 요약

Discord/Slack에 설치하면 AI가 커뮤니티의 톤앤매너를 학습하여:
1. 개인화된 웰컴 메시지 자동 전송
2. FAQ 및 반복 질문 자동 응답
3. 미응답 질문 감지 및 운영자 알림
4. 이탈 위험 멤버 조기 예측 및 리인게이지

설치 5분, 학습 24시간, 완전 자동화.

### 1.3 성공 지표 (수익 중심)

| 지표 | 목표 (4개월) | 측정 방법 |
|------|-------------|-----------|
| MRR | $12,000 | Stripe 대시보드 |
| 유료 고객 수 | 150+ | 활성 구독자 수 |
| 봇 설치 수 | 500+ | Discord/Slack OAuth |
| 자동화 메시지 | 50K+/월 | 봇 활동 로그 |
| 평균 응답 시간 | <30초 | FAQ 응답 latency |
| 고객 이탈률 | <5%/월 | 구독 해지율 |

## 2. 타겟 사용자

### 2.1 주요 페르소나 (지불자)

**페르소나 1: 유료 커뮤니티 운영자 "민수"**
- 나이/직업: 35세, 코딩 부트캠프 운영자
- 상황: 월 $99 유료 멤버십, 회원 500명, Discord 운영
- 현재 문제:
  - 새 멤버 온보딩에 주 15시간 소모
  - 같은 질문 반복 응답 (스트레스)
  - 파트타임 VA 고용 비용 $800/월
- Pain Point: "커뮤니티 관리가 본업이 되어버렸어요"
- 지불 의향: $99/월 (VA 비용의 1/8)
- 기대 효과: 주 15시간 -> 2시간으로 단축

**페르소나 2: SaaS 커뮤니티 매니저 "지영"**
- 나이/직업: 30세, B2B SaaS 회사 커뮤니티 매니저
- 상황: 고객 커뮤니티 2,000명, Slack 운영, 이탈 방지가 KPI
- 현재 문제:
  - 미응답 질문으로 고객 불만 -> 이탈 연결
  - 비활성 고객 조기 감지 불가
  - 수동 분석에 주 10시간 소모
- Pain Point: "이탈 징후를 놓쳐서 갱신율이 떨어졌어요"
- 회사 도구 예산: 인당 $200/월
- 지불 의향: $199/월 (여러 채널 관리 필요)

**페르소나 3: 크리에이터 "Jessica"**
- 나이/직업: 28세, YouTube 크리에이터 (구독자 50만)
- 상황: Patreon 연동 Discord, 후원자 3,000명
- 현재 문제:
  - 새 후원자 환영을 일일이 못함
  - 후원 등급별 차별화된 응대 어려움
  - 콘텐츠 제작에 집중하고 싶음
- Pain Point: "팬들에게 미안한데 관리할 시간이 없어요"
- 지불 의향: $49/월 (월 후원 수익 $10K의 0.5%)

### 2.2 사용자 여정 (User Journey)

```
[인지] -> [탐색] -> [설치] -> [설정] -> [학습] -> [자동화] -> [확장]

1. 인지: "커뮤니티 관리 자동화" 검색 or 동료 추천
2. 탐색: 랜딩 페이지에서 데모 영상 시청 (2분)
3. 설치: Discord/Slack OAuth로 1-click 설치 (30초)
4. 설정: 대시보드에서 기본 설정 (5분)
5. 학습: AI가 지난 30일 대화 분석 (24시간)
6. 자동화: 웰컴/FAQ/알림 자동 동작 시작
7. 확장: 추가 커뮤니티 연결, 고급 기능 활성화
```

### 2.3 핵심 사용 시나리오

**시나리오 1: 자동 웰컴 + 온보딩 시퀀스**
1. 새 멤버 "Tom"이 민수의 유료 커뮤니티에 가입
2. CommunityBot이 즉시 DM 전송:
   - "Tom님, 코딩 부트캠프에 오신 것을 환영합니다!"
   - 커뮤니티 톤앤매너에 맞춘 자연스러운 문체
3. 자기소개 채널 안내 + 첫 주 로드맵 제공
4. 24시간 후 자기소개 미작성 시 부드러운 리마인더
5. 7일 후 첫 주 완료 축하 + 다음 단계 안내
6. 민수는 아무것도 안 해도 온보딩 완료율 70% -> 95%

**시나리오 2: FAQ 자동 응답**
1. 멤버가 #질문 채널에 "강의 영상 다운로드 가능한가요?" 질문
2. CommunityBot이 유사 질문 히스토리 분석 (300ms)
3. 자동 응답: "안녕하세요! 다운로드는 Premium 플랜에서 가능해요. 업그레이드는 여기서..."
4. 정확도 신뢰도가 낮으면 응답 보류 + 운영자에게 알림
5. 운영자 응답을 학습 데이터에 추가 (점점 더 정확해짐)

**시나리오 3: 미응답 질문 알림**
1. 멤버 "Sarah"가 일요일 새벽 3시에 기술 질문 작성
2. 4시간 경과, 아무 응답 없음
3. CommunityBot이 운영자에게 DM 알림:
   - "미응답 질문 감지: Sarah님의 질문이 4시간째 답변 대기 중입니다"
   - 질문 링크 + 예상 답변 초안 제공
4. 운영자가 답변하거나, AI 초안 승인으로 빠르게 처리
5. Sarah의 불만 -> 감사로 전환

**시나리오 4: 이탈 예측 및 리인게이지**
1. CommunityBot이 멤버 활동 패턴 분석
2. "김철수"님의 이탈 위험도 85%로 감지:
   - 접속 빈도: 매일 -> 주 1회
   - 채팅 참여: 10회/주 -> 0회
   - 리액션: 감소 추세
3. 운영자 대시보드에 경고 표시
4. 자동 리인게이지 DM 발송:
   - "철수님, 요즘 바쁘신가봐요! 이번 주 인기 콘텐츠 모아봤어요"
   - 개인화된 콘텐츠 추천
5. 이탈 예방 성공률 40% 달성

## 3. 기능 요구사항

### 3.1 MVP 필수 기능 (3개)

**MVP 1: AI 웰컴 시퀀스**
- 새 멤버 가입 감지 시 자동 DM 전송
- 커뮤니티 톤앤매너 학습 기반 자연스러운 문체
- 커스텀 템플릿 (변수: {name}, {server}, {role})
- 지연 메시지 시퀀스 (즉시, 24h, 48h, 7d)
- 조건부 메시지 (자기소개 미작성 시 리마인더)
- 역할 기반 차별화 (Patron tier별 다른 웰컴)

**MVP 2: FAQ 자동 응답 (AI 기반)**
- 지난 30일 대화 분석으로 FAQ 자동 추출
- 유사 질문 감지 시 자동 응답 (임계값 조절 가능)
- 신뢰도 낮은 응답은 보류 + 운영자 확인 요청
- 운영자 응답 학습 -> 점진적 정확도 향상
- 커뮤니티별 독립 학습 모델
- 멀티 언어 지원 (한/영/일)

**MVP 3: 미응답 질문 알림**
- 채널별 미응답 시간 임계값 설정 (기본 4시간)
- 미응답 질문 감지 시 운영자 DM/채널 알림
- 알림에 AI 답변 초안 포함
- 미응답 질문 대시보드 (우선순위 정렬)
- 응답 완료 자동 감지 및 알림 해제

### 3.2 Phase 2 기능 (향후)

- **이탈 예측 엔진**: 활동 패턴 기반 이탈 위험도 스코어링
- **자동 리인게이지**: 이탈 위험 멤버에게 개인화 DM
- **Slack 지원**: Discord 우선 후 Slack 확장
- **분석 대시보드**: 멤버 활동 지표, 응답률, 리텐션
- **Zapier/Make 연동**: 외부 워크플로우 자동화
- **화이트라벨**: 에이전시용 브랜드 커스터마이징
- **API 제공**: 커스텀 통합 개발용

### 3.3 명시적 제외 (Out of Scope)

- **모더레이션**: 스팸 차단, 욕설 필터링 (MEE6, Dyno 영역)
- **티켓 시스템**: 고객 지원 티켓 관리 (Zendesk, Intercom 영역)
- **음성 채널**: 음성/화상 관련 기능
- **결제/멤버십**: 구독 관리는 Patreon, Circle 영역
- **콘텐츠 호스팅**: 강의/콘텐츠 자체 호스팅
- **범용 챗봇**: ChatGPT 스타일 자유 대화

## 4. 기술 요구사항

### 4.1 시스템 아키텍처

```
[Discord/Slack API] <--> [Bot Gateway (Node.js)]
                               |
                               v
                    [Event Handler Service]
                               |
         ------------------------------------------------
         |                |                |             |
         v                v                v             v
   [Welcome         [FAQ Engine]      [Unanswered    [Churn
    Sequence]                          Monitor]       Predictor]
         |                |                |             |
         v                v                v             v
                   [AI Service Layer (Claude API)]
                               |
         ------------------------------------------------
         |                |                |             |
         v                v                v             v
   [Tone/Manner      [Semantic        [Activity      [Risk
    Learner]          Search]          Tracker]       Scorer]
                               |
                               v
                    [Message Queue (Redis)]
                               |
                               v
                    [Rate Limiter + Scheduler]
                               |
                               v
                    [Discord/Slack API Response]
```

### 4.2 기술 스택

| 레이어 | 기술 | 선정 이유 |
|--------|------|-----------|
| Bot Runtime | Node.js + discord.js / Slack Bolt | 공식 라이브러리, 안정성 |
| API Server | Hono on Cloudflare Workers | 서버리스, 글로벌 엣지 |
| AI Engine | Claude 3.5 Sonnet API | 톤앤매너 학습, 자연어 이해 |
| Vector DB | Pinecone | FAQ 유사도 검색 |
| Database | PlanetScale (MySQL) | 서버리스, 무료 티어 |
| Cache/Queue | Upstash Redis | 메시지 큐, 레이트리밋 |
| Scheduler | Upstash QStash | 시퀀스 메시지 스케줄링 |
| 결제 | Stripe | 구독 관리, 글로벌 결제 |
| 대시보드 | Next.js + Tailwind | SPA, 빠른 개발 |
| 모니터링 | Sentry + Axiom | 에러 추적, 로그 분석 |

### 4.3 AI/ML 컴포넌트

**톤앤매너 학습 (Tone Learning)**
- 커뮤니티 과거 메시지 분석 (운영자/모더레이터 메시지 우선)
- Few-shot learning으로 스타일 벡터 생성
- 메시지 생성 시 스타일 가이드로 활용

**FAQ 시맨틱 검색**
- 질문 -> 임베딩 변환 (text-embedding-3-small)
- Pinecone에서 유사 질문 검색 (cosine similarity > 0.85)
- Claude로 답변 생성 또는 기존 답변 반환

**이탈 예측 모델**
- Features: 접속 빈도, 채팅 수, 리액션 수, 응답률 변화
- 간단한 로지스틱 회귀 (초기) -> 점진적 고도화
- 위험도 스코어: 0-100 (80+ = 고위험)

### 4.4 외부 의존성

| 의존성 | 용도 | 월 비용 (예상) |

async function handleQuestion(message, communityId) {
  // 1. 질문 임베딩 생성
  const embedding = await openai.embeddings.create({
    model: "text-embedding-3-small",
    input: message.content
  });

  // 2. 유사 FAQ 검색
  const similar = await pinecone.query({
    vector: embedding.data[0].embedding,
    topK: 3,
    filter: { communityId }
  });

  // 3. 신뢰도 체크
  if (similar.matches[0]?.score > 0.85) {
    // 자동 응답
    const response = await generateResponse(
      message.content,
      similar.matches[0].metadata.answer,
      communityTone[communityId]
    );
    return { action: "auto_reply", response };
  } else if (similar.matches[0]?.score > 0.7) {
    // 운영자 확인 요청
    return {
      action: "request_approval",
      draft: await generateDraft(message.content, similar),
      confidence: similar.matches[0].score
    };
  }

  // 4. 새 질문 - 학습 대기
  return { action: "learn", questionId: message.id };
}
