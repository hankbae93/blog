## 1. 개요

### 1.1 문제 정의 (Problem Statement)

- **누가** 겪는 문제인가?
  - 여러 프로젝트를 동시에 진행하는 개발자
  - 컨텍스트 스위칭이 잦은 스타트업 개발자
  - 사이드 프로젝트를 저녁/주말에만 진행하는 개발자
  - 과거 결정의 맥락을 팀에게 설명해야 하는 테크 리드

- **무엇이** 문제인가?
  - 프로젝트 전환 시 "어디까지 했더라?"를 기억해내는 데 30-60분 소비
  - "왜 이렇게 구현했더라?"는 커밋 메시지, 코멘트 어디에도 없음
  - IDE는 파일만 저장하고, Git은 변경사항만 저장하고, 노트앱은 코드와 분리됨
  - 개발자의 "사고 맥락(thinking context)"이 어디에도 저장되지 않음

- **왜** 문제인가? (전제 파괴 관점)
  - **파괴해야 할 전제:** "코드와 커밋 메시지만 남기면 나중에 이해할 수 있다"
  - **실제 현실:** 코드는 "무엇(what)"을 말해주지만, "왜(why)"와 "그때 내가 뭘 고민했는지(context)"는 사라진다. 커밋 메시지는 너무 짧고, 주석은 코드와 함께 진화하지 않고, Notion/노트앱은 코드와 물리적으로 분리되어 있다.
  - AI 시대에 개발자에게 남는 진짜 가치는 **"왜 이렇게 만들었는지 기억하고 설명하는 능력"**이다. 코드 생성은 AI가 하지만, 의사결정의 맥락은 인간만이 보존할 수 있다.

### 1.2 솔루션 개요

DevContext는 개발자의 사고 맥락을 코드와 함께 저장하고 복원하는 도구이다.
- IDE 활동 + Git 변경 + 개발자 노트를 하나의 "컨텍스트 스냅샷"으로 통합
- 프로젝트 재진입 시 "어디서 멈췄는지"를 자동으로 복원
- "왜 이렇게 했는지"를 코드 라인 단위로 기록하고 검색

### 1.3 성공 지표 (KPIs)

1. **컨텍스트 복원 시간 단축**: 프로젝트 전환 후 작업 재개까지 걸리는 시간 50% 감소 (30분 -> 15분)
2. **컨텍스트 스냅샷 생성률**: 주간 활성 사용자의 80%가 최소 3개 이상의 스냅샷 생성
3. **주간 활성 사용자(WAU)**: 런치 4주 후 2,000명 달성

## 2. 타겟 사용자

### 2.1 Primary Persona

**페르소나 1: 멀티태스킹 개발자 - 준혁 (30세, 풀스택 개발자)**
- 회사 프로젝트 2개 + 사이드 프로젝트 1개 동시 진행
- 월요일 아침마다 "금요일에 뭐 하다 말았더라?" 30분 헤매기
- 커밋 로그를 뒤지고, Slack을 검색하고, 노션을 열어보지만 맥락이 안 잡힘
- "내가 왜 이렇게 짰는지 모르겠다"며 코드를 다시 읽는 데 시간 낭비

**페르소나 2: 주말 빌더 - 소영 (28세, 백엔드 개발자, 사이드 프로젝트 운영)**
- 평일은 회사 일, 주말에만 사이드 프로젝트
- 2주 만에 프로젝트 열면 "아, 이게 뭐였지?" 처음부터 다시 이해해야 함
- 노션에 개발 일지를 쓰지만 코드와 동기화 안 됨
- "기억력이 문제가 아니라 도구가 없는 거다"라고 느낌

**페르소나 3: 설명해야 하는 테크 리드 - 민재 (35세, 스타트업 CTO)**
- 6개월 전 아키텍처 결정 이유를 신입 개발자에게 설명해야 함
- PR 코멘트, 커밋 메시지, Slack 검색해도 "왜"는 안 나옴
- "그때 왜 이렇게 했는지 문서가 없네" 매번 반복
- 의사결정 맥락이 사라지면 팀 온보딩도 느려짐

### 2.2 User Journey

```
1. 개발 세션 시작
   └─> DevContext가 현재 상태(열린 파일, Git 브랜치, 최근 변경) 자동 감지

2. 개발 중
   └─> 특정 코드에 대한 생각/이유를 인라인으로 기록
   └─> "이 함수는 나중에 리팩토링 필요" 같은 TODO+이유 기록

3. 세션 종료
   └─> 자동 또는 수동으로 "컨텍스트 스냅샷" 저장
   └─> 현재 상태 + 진행 상황 + 다음 할 일 요약

4. 며칠 후 프로젝트 재진입
   └─> DevContext가 마지막 스냅샷 표시
   └─> "여기서 멈췄고, 이게 다음 할 일이었어요"
   └─> 30분 헤매던 시간이 5분으로 단축

5. 과거 결정 검색
   └─> "왜 Redis 대신 PostgreSQL 썼지?" 검색
   └─> 해당 시점의 컨텍스트 + 기록된 이유 표시
```

### 2.3 Use Case Scenarios

**시나리오 1: 월요일 아침 복귀**
1. 준혁이 금요일에 작업하던 프로젝트를 월요일에 연다
2. DevContext가 마지막 세션 요약을 표시: "인증 모듈 작업 중, 토큰 갱신 로직 구현 완료, 다음: 에러 핸들링 추가"
3. 금요일에 기록한 노트: "갱신 실패 시 로그아웃 vs 재시도? -> 재시도 3회 후 로그아웃으로 결정 (UX팀 협의)"
4. 준혁은 5분 만에 작업에 복귀, 바로 에러 핸들링 코드 작성 시작

**시나리오 2: 사이드 프로젝트 2주 만에 재개**
1. 소영이 2주 만에 사이드 프로젝트 폴더를 연다
2. DevContext가 타임라인 표시: 마지막 세션들의 스냅샷 목록
3. 가장 최근 스냅샷: "결제 연동 중, Stripe webhook 테스트까지 완료"
4. 코드에 남긴 컨텍스트 노트: "webhook signature 검증 - 공식 문서 예제 따름, 커스텀 로직 필요하면 여기 수정"
5. 소영은 코드 전체를 다시 읽을 필요 없이 바로 이어서 작업

**시나리오 3: 신입 온보딩 설명**
1. 민재가 6개월 전 DB 설계 결정을 신입에게 설명해야 함
2. DevContext에서 "database schema" 검색
3. 당시 컨텍스트 스냅샷 발견: "NoSQL vs RDB 고민 -> 트랜잭션 필요해서 PostgreSQL 선택"
4. 추가 노트: "처음엔 MongoDB 고려했으나 결제 데이터 정합성 때문에 RDB로 결정"
5. 민재는 기억에 의존하지 않고 기록된 맥락으로 설명

## 3. 기능 요구사항

### 3.1 MVP 핵심 기능 (3개)

**기능 1: 컨텍스트 스냅샷**
- 현재 IDE 상태(열린 파일, 커서 위치, Git 브랜치)를 스냅샷으로 저장
- 세션 종료 시 자동 저장 또는 수동 저장 (Cmd+Shift+S)
- 스냅샷에 "지금 뭘 하고 있었는지" 한 줄 요약 추가
- 본질적 가치: **"어디서 멈췄는지"를 기억에 의존하지 않게 함**

**기능 2: 인라인 컨텍스트 노트**
- 코드 라인에 "왜 이렇게 했는지" 노트 연결
- Git blame처럼 보이지만, 코드가 아니라 "이유"가 표시됨
- 노트는 코드와 함께 이동 (코드 변경 시 노트도 같이 이동)
- 검색 가능: "왜 async 썼지?" -> 해당 노트로 이동
- 본질적 가치: **코드와 이유를 물리적으로 같은 곳에 저장**

**기능 3: 프로젝트 재진입 대시보드**
- 프로젝트 열 때 자동으로 마지막 컨텍스트 표시
- 최근 스냅샷 요약, 미완료 TODO, 다음 할 일 제안
- "지난 세션에서 변경한 파일" 목록
- 타임라인 뷰로 과거 세션 브라우징
- 본질적 가치: **컨텍스트 복원 시간을 극적으로 단축**

### 3.2 향후 기능 (Phase 2)

- AI 기반 컨텍스트 요약: 코드 변경사항을 자동으로 요약
- 팀 컨텍스트 공유: PR에 컨텍스트 노트 자동 첨부
- 음성 메모: 코딩 중 음성으로 생각 기록
- Git 커밋과 연동: 커밋 시 관련 컨텍스트 노트 자동 참조
- 의사결정 로그: "A vs B" 결정 기록 및 추적

### 3.3 명시적 제외 (Out of Scope)

- 코드 에디터 기능 (VS Code 영역)
- Git 클라이언트 기능 (GitKraken, Fork 영역)
- 프로젝트 관리 기능 (Linear, Jira 영역)
- 코드 리뷰 기능 (GitHub PR 영역)
- 범용 노트 앱 기능 (Notion, Obsidian 영역)

## 4. 기술 요구사항

### 4.1 아키텍처 개요

```
[VS Code Extension]           [Desktop App (Electron)]          [Cloud Sync]
       |                              |                              |
       v                              v                              v
+----------------+            +----------------+            +----------------+
| Context        |<---------->| Core Engine    |<---------->| Sync Service   |
| Capture        |            | (Rust)         |            | (Optional)     |
+----------------+            +----------------+            +----------------+
       |                              |                              |
       v                              v                              v
+----------------+            +----------------+            +----------------+
| Inline Notes   |            | Local DB       |            | Cloud Storage  |
| UI             |            | (SQLite)       |            | (Supabase)     |
+----------------+            +----------------+            +----------------+
                                      |
                                      v
                              +----------------+
                              | Git Integration|
                              | (libgit2)      |
                              +----------------+
```

### 4.2 기술 스택 제안

**VS Code Extension (Primary)**
- Language: TypeScript
- VS Code Extension API
- SQLite (better-sqlite3) for local storage

**Core Engine**
- Language: Rust (성능, 크로스 플랫폼)
- SQLite for persistent storage
- libgit2 for Git integration
- Tauri (optional desktop app)

**Cloud Sync (Optional)**
- Backend: Supabase (PostgreSQL + Auth)
- Real-time: Supabase Realtime
- Storage: Supabase Storage (첨부 파일)

**Analytics & Monitoring**
- PostHog (제품 분석)
- Sentry (에러 트래킹)

### 4.3 인프라 요구사항

**로컬 우선 아키텍처**
- 모든 데이터는 로컬에 먼저 저장 (오프라인 지원)
- 클라우드 동기화는 선택 사항
- 프라이버시: 코드/노트가 강제로 클라우드에 올라가지 않음

**스토리지 요구사항**
- 프로젝트당 예상 데이터: ~10MB (스냅샷 + 노트)
- SQLite 단일 파일로 관리
- Git 저장소 내 `.devcontext/` 폴더 (선택적 커밋 가능)

**성능 요구사항**
- 스냅샷 저장: < 100ms
- 프로젝트 로드 시 컨텍스트 표시: < 500ms
- 노트 검색: < 200ms (10,000개 노트 기준)

## 5. 비즈니스 요구사항

### 5.1 수익 모델

**Freemium 모델**

| 플랜 | 가격 | 기능 |
|------|------|------|
| Free | $0 | 프로젝트 3개, 로컬 저장만, 기본 스냅샷 |
| Pro | $9/월 | 무제한 프로젝트, 클라우드 동기화, AI 요약, 검색 고급 기능 |
| Team | $19/월/인 | Pro 전체 + 팀 컨텍스트 공유, 권한 관리, 감사 로그 |

**가격 설정 근거**
- VS Code 유료 확장 시장 평균 가격대
- 생산성 도구(Notion, Linear)보다 낮게, 진입 장벽 낮춤
- "매일 30분 아끼면 $9 아깝지 않다" 포지셔닝

### 5.2 시장 규모 추정

**TAM (Total Addressable Market)**
- 전 세계 소프트웨어 개발자: ~2,700만 명
- VS Code 사용자: ~1,500만 명 (약 55%)

**SAM (Serviceable Addressable Market)**
- 컨텍스트 스위칭 문제를 겪는 개발자 (멀티 프로젝트): ~40%
- 대상: 600만 명

**SOM (Serviceable Obtainable Market)**
- 1년 내 목표: 50,000 설치, 5,000 유료 전환
- 연간 매출 목표: $540K ARR (5,000 x $9 x 12)

**경쟁 환경**
| 도구 | 포지션 | DevContext 차별점 |
|------|--------|-------------------|
| VS Code Workspaces | 파일/창 상태만 저장 | 사고 맥락(왜?)까지 저장 |
| Git commit messages | 변경 사항만 기록 | 결정 이유, 컨텍스트 기록 |
| Notion/Obsidian | 코드와 물리적 분리 | 코드와 노트 동일 공간 |
| GitHub Copilot | 코드 생성 | 컨텍스트 보존 (상호 보완) |
