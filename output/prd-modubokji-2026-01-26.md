# PRD: 모두복지 Pro

**한국 복지/지원금 AI 어시스턴트**

---

## 1. 개요

### 1.1 문제 정의

한국의 복지 제도는 중앙정부, 지자체, 공공기관별로 **수백 개의 지원금과 혜택**이 파편화되어 있다. 복지를 필요로 하는 사람들이 다음과 같은 문제를 겪고 있다:

**핵심 문제:**
- **발견의 어려움**: "내가 받을 수 있는 복지가 뭔지 모른다"
- **자격 확인의 복잡성**: "조건이 너무 복잡해서 해당되는지 모르겠다"
- **신청 장벽**: "서류가 뭐가 필요한지 몰라서 주민센터 여러 번 방문"
- **타이밍 실패**: "신청 기간 놓쳐서 혜택 못 받음"

**방치된 이유:**
- 복지 제도가 부처별로 파편화
- 매년 제도 변경/신설/폐지 반복
- 수익성 낮아 스타트업이 안 건드림
- 정부 시스템(복지로, 정부24)은 UX 열악

### 1.2 솔루션

**모두복지 Pro**는 AI 기반으로 사용자의 상황을 분석하여:
1. 받을 수 있는 복지를 **자동으로 추천**하고
2. 자격 조건을 **쉽게 확인**하고
3. 필요 서류를 **체크리스트로 정리**하고
4. 신청 마감일을 **알림**으로 놓치지 않게 한다

### 1.3 성공 지표 (MVP 기준)

| 지표 | 목표 | 측정 방법 |
|------|------|----------|
| 월간 활성 사용자 | 1,000명 | 월 1회 이상 조회 |
| 복지 매칭 정확도 | 90%+ | 사용자 피드백 |
| 신청 전환율 | 30%+ | 추천 → 신청 클릭 |
| NPS | 50+ | 월간 설문 |

## 2. 타겟 사용자

### 2.1 Primary Persona: 김복지 (52세, 1인 사업자)

**배경:**
- 동네 분식집 운영 (월 매출 800만원)
- 스마트폰 사용에 익숙하지 않음
- 코로나 때 지원금 받아봤지만 과정이 힘들었음

**Pain Points:**
- "사업자도 받을 수 있는 게 있다던데... 뭔지 모르겠어"
- "주민센터 갔더니 서류 부족하다고 다시 오래"
- "마감 넘겨서 못 받은 거 한두 번이 아냐"

**행동 패턴:**
- 카카오톡으로 정보 공유받음
- 네이버 카페에서 복지 정보 검색
- 복잡하면 그냥 포기

### 2.2 Secondary Persona: 이취준 (28세, 프리랜서)

**배경:**
- 디자인 프리랜서, 월 소득 불규칙 (200~400만원)
- 4대보험 미가입
- 청년 지원금 정보에 관심 많음

**Pain Points:**
- "프리랜서라 뭐가 해당되는지 애매해"
- "소득 기준이 매번 달라서 헷갈려"
- "조건 충족하는 줄 알고 신청했다가 탈락"

### 2.3 사용 시나리오

**시나리오 1: 김복지의 복지 찾기**
1. 카카오톡에서 "1인 사업자 지원금" 기사 클릭
2. 모두복지 Pro 접속
3. 간단한 정보 입력 (나이, 소득, 사업자 여부)
4. AI가 "소상공인 경영안정자금" 등 3개 추천
5. 각 복지 상세 정보 + 필요 서류 확인
6. "마감 7일 전 알림" 설정

**시나리오 2: 이취준의 자격 확인**
1. "청년내일저축계좌 자격되나?" 검색
2. 모두복지 Pro에서 자격 시뮬레이션
3. 소득 기준 입력 → "조건 충족" 결과
4. 필요 서류 체크리스트 저장
5. 신청 링크로 이동

## 3. 기능 요구사항

### 3.1 MVP 기능 (v1.0)

#### F1. 자격조건 자동 체크
**설명:** 사용자 정보 기반으로 복지 자격 자동 판단
- 입력: 나이, 지역, 소득, 가구원 수, 사업자 여부
- 출력: 자격 충족 여부 + 근거 조항
- AI: 조건 해석 및 경계 케이스 판단

**수용 기준:**
- [ ] 5개 이상 복지 제도에 대해 자격 판단 가능
- [ ] 판단 근거를 조항과 함께 제시
- [ ] 90% 이상 정확도 (수동 검증 기준)

#### F2. 필요 서류 체크리스트
**설명:** 선택한 복지의 필요 서류를 체크리스트로 제공
- 서류명 + 발급처 + 온라인 발급 가능 여부
- 체크박스로 준비 현황 관리
- PDF 다운로드/공유 기능

**수용 기준:**
- [ ] 주요 5개 복지에 대한 서류 정보 완비
- [ ] 온라인 발급 링크 연결
- [ ] 체크리스트 상태 저장

#### F3. 마감일 알림
**설명:** 관심 복지의 신청 마감일 알림
- 7일 전, 3일 전, 당일 알림
- 카카오톡 알림톡 or 앱 푸시

**수용 기준:**
- [ ] 3개 이상 복지에 대해 마감일 데이터 보유
- [ ] 알림 수신 동의 시 발송
- [ ] 알림 클릭 시 해당 복지 상세페이지 이동

### 3.2 향후 기능 (v2.0+)

| 우선순위 | 기능 | 설명 |
|---------|------|------|
| High | 대화형 상담 | AI 챗봇으로 복잡한 상황 상담 |
| High | 신청서 작성 도우미 | 신청서 양식 자동 채우기 |
| Medium | 복지 히스토리 | 과거 받은 복지 기록 관리 |
| Medium | 가족 관리 | 가구원 전체 복지 통합 관리 |
| Low | 지자체 연동 | 지역별 복지 데이터 자동 수집 |

### 3.3 명시적 제외 범위

- **세금 신고 대행**: 법적 이슈
- **대출 상담**: 금융 규제 대상
- **의료 상담**: 의료법 위반 가능성
- **신청 대행**: 대리 신청은 제공하지 않음

## 5. 비즈니스 요구사항

### 5.1 수익 모델

| 티어 | 가격 | 기능 |
|------|------|------|
| Free | 무료 | 월 3회 조회, 기본 추천 |
| Pro | 월 9,900원 | 무제한 조회, 알림, 서류 가이드 |
| Family | 월 14,900원 | 가구원 5명까지 관리 |

**예상 수익 (12개월 후):**
- MAU 10,000명 가정
- 유료 전환율 5% = 500명
- Pro 평균 = 월 500만원
- 연 매출 = 6,000만원

### 5.2 시장 규모

**TAM (Total Addressable Market):**
- 복지 사각지대 인구: 약 500만 명
- 잠재 시장: 500만 × 10,000원 = 5,000억원/년

**SAM (Serviceable Available Market):**
- 디지털 접근 가능 인구: 약 200만 명
- 접근 가능 시장: 200만 × 10,000원 = 2,000억원/년

**SOM (Serviceable Obtainable Market):**
- 1년 내 목표: 1만 명 × 10,000원 × 5% = 5,000만원/년

### 5.3 GTM (Go-to-Market) 전략

**Phase 1: 검증 (Month 1-2)**
- 타겟: 1인 사업자 커뮤니티 (네이버 카페)
- 채널: 무료 복지 진단 이벤트
- 목표: 500명 얼리어답터

**Phase 2: 성장 (Month 3-6)**
- SEO: "소상공인 지원금", "청년 복지" 키워드
- 콘텐츠: 복지 정보 블로그/유튜브
- 파트너십: 세무사/노무사 협회

**Phase 3: 확장 (Month 7-12)**
- B2B: 기업 복리후생 서비스 연계
- B2G: 지자체 복지 안내 시스템 제안
