## 1. 개요

### 1.1 문제 정의

- **누가** 겪는 문제인가?
  - 회의 녹음을 텍스트로 정리하고 싶은 Mac 유저
  - 민감한 대화(법률 상담, 의료 상담)를 기록하는 전문직
  - 아이디어 메모를 음성으로 빠르게 남기는 크리에이터

- **무엇이** 문제인가?
  - 회의 녹음했는데 다시 듣기 귀찮음 (시간 소모)
  - Otter.ai 같은 서비스를 쓰고 싶지만 민감한 내용이라 클라우드 업로드가 꺼려짐 (프라이버시)
  - 텍스트 변환 후 정리하는 것이 또 다른 노동 (추가 작업)

- **왜** 문제인가? (전제 파괴 관점)
  - **파괴해야 할 전제:** "좋은 STT(Speech-to-Text)는 클라우드에서만 가능하다"
  - **실제 현실:** Apple MLX 프레임워크의 성숙과 mlx-audio 라이브러리(2026년 1월 1,233 stars)의 등장으로, 로컬에서도 클라우드 수준의 STT 품질이 가능해졌다.
  - 민감한 회의를 녹음만 하고 정리 안 하거나, 프라이버시 걱정 때문에 수동으로 타이핑하는 비합리적 행동이 만연하다.

### 1.2 솔루션 요약

VoiceMemo Pro는 Apple Silicon Mac에서 100% 로컬로 동작하는 음성 메모 앱이다.
- mlx-audio 기반 로컬 STT로 녹음 즉시 텍스트 변환
- 변환된 텍스트를 자동으로 구조화 (요약, 할일, 키워드)
- Apple Notes/Obsidian으로 원클릭 내보내기

### 1.3 성공 지표 (최대 3개)

1. **일간 녹음 수**: 활성 사용자 평균 일 2회 이상 녹음
2. **프라이버시 만족도**: NPS 설문에서 "데이터 프라이버시" 항목 > 90점
3. **유료 전환율**: 무료 체험 -> 유료 전환율 > 8%

## 2. 타겟 사용자

**페르소나 1: 프라이버시 의식 전문직 - 수진 (38세, 변호사)**

- 현재는 수기로 메모하거나 녹음 후 직접 듣고 타이핑
- MacBook Pro M3 사용 중

**페르소나 2: 아이디어 캡처러 - 준영 (29세, UX 디자이너)**
- 이동 중이나 샤워 중에 아이디어가 떠오름
- 빠르게 음성으로 기록하고 싶지만, 나중에 정리가 귀찮음
- Otter.ai를 써봤지만 개인적인 아이디어를 클라우드에 올리기 꺼림
- Mac Studio + iPhone 연동 원함

**페르소나 3: 회의 정리 피로자 - 혜원 (33세, PM)**
- 하루 3-4개 회의, 모두 녹음하지만 다시 듣지 않음
- 회의록 작성에 시간을 많이 씀
- 회사 보안 정책상 외부 SaaS 사용 제한
- "녹음만 하고 정리 안 하는" 자신에게 죄책감

### 2.2 사용자 시나리오

**시나리오 1: 법률 상담 기록**
1. 수진이 의뢰인과 30분 상담 시작
2. VoiceMemo Pro로 녹음 버튼 클릭
3. 상담 종료 후, 앱이 자동으로 텍스트 변환 (로컬 처리)
4. 핵심 요약 + 법적 쟁점 + 다음 단계가 자동 구조화
5. Obsidian의 의뢰인 폴더로 내보내기
6. 녹음 원본과 텍스트 모두 로컬 저장, 클라우드 전송 없음

**시나리오 2: 이동 중 아이디어 캡처**
1. 준영이 산책 중 앱 아이디어가 떠오름
2. Apple Watch에서 VoiceMemo Pro 시작 (향후 기능)
3. 2분간 생각을 말로 기록
4. Mac에 돌아오면 자동 동기화 (로컬 네트워크)
5. 텍스트 + 키워드 태그가 정리되어 있음

**시나리오 3: 팀 회의 정리**
1. 혜원이 1시간 팀 미팅 녹음
2. 회의 종료 후 "구조화" 버튼 클릭
3. 논의 주제별 요약 + 결정 사항 + 액션 아이템 생성
4. 회사 Notion에 복사/붙여넣기
5. 회의록 작성 시간 30분 -> 5분으로 단축

## 3. 기능 요구사항

### 3.1 MVP 필수 기능 (최대 3개)

**기능 1: 로컬 음성 녹음 + mlx-audio STT 변환**
- 고품질 음성 녹음 (WAV/M4A)
- mlx-audio Whisper 기반 로컬 STT
- 실시간 진행률 표시
- 본질적 가치: 프라이버시 - 데이터가 기기를 떠나지 않음

**기능 2: 변환 텍스트 자동 구조화**
- 로컬 LLM(mlx-lm) 기반 텍스트 분석
- 자동 생성: 요약 (3문장), 할일/액션 아이템, 키워드 태그
- 사용자 수정 가능
- 본질적 가치: 정리 노동 제거 - 녹음만 하면 노트가 완성

**기능 3: Apple Notes/Obsidian 내보내기**
- 마크다운 포맷 내보내기
- Apple Notes 직접 연동 (AppleScript)
- Obsidian 볼트 폴더 직접 저장
- 본질적 가치: 워크플로우 통합 - 기존 도구와 자연스러운 연결

## 4. 기술 요구사항

### 4.1 시스템 아키텍처

```
[macOS App - SwiftUI]
        |
        v
+------------------+      +------------------+
|   Audio Engine   | ---> |   mlx-audio STT  |
|   (AVFoundation) |      |   (Whisper)      |
+------------------+      +------------------+
                                  |
                                  v
                          +------------------+
                          |   mlx-lm (LLM)   |
                          |   Structuring    |
                          +------------------+
                                  |
                                  v
                          +------------------+
                          |   Export Engine  |
                          | (Notes/Obsidian) |
                          +------------------+
```

### 4.2 기술 스택

**Core**
- Platform: macOS 14+ (Apple Silicon 전용)
- Language: Swift 5.9
- UI: SwiftUI + AppKit (하이브리드)
- Audio: AVFoundation

**ML/AI**
- STT: mlx-audio (Whisper base/small)
- LLM: mlx-lm (Llama 3 8B 또는 Phi-3)
- Framework: Apple MLX

**Storage**
- Local DB: Core Data 또는 SwiftData
- File: 로컬 파일 시스템 (~/Documents/VoiceMemoPro/)

### 4.3 외부 의존성

- **mlx-audio**: Speech-to-Text 엔진 (MIT 라이선스)
- **mlx-lm**: 로컬 LLM 추론 (Apache 2.0)
- **Whisper 모델**: OpenAI Whisper weights (MIT)
- **LLM 모델**: Llama 3 (Meta 라이선스) 또는 Phi-3 (MIT)

**의존성 리스크**
- MLX 프레임워크가 Apple 공식이므로 안정적
- 모델 weights는 앱 번들 또는 첫 실행 시 다운로드
