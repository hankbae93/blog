## 1. 주간 트렌드 TOP 5

### 1. AI 에이전트 "실행 시대" — 대화에서 작업 수행으로 전환 완료 🔥 Rising
**이번 주 등장 횟수**: 6회 (매일)
**관련 제품/기사**:
- [Jupid](https://www.producthunt.com/products/jupid) (510↑) — AI가 은행 거래를 IRS 카테고리로 자동 분류, ~96% 정확도
- [Influcio](https://www.producthunt.com/products/influcio-2) (462↑) — AI가 인플루언서 매칭→캠페인 집행→성과 학습까지 자율 수행
- [Notion MCP](https://www.producthunt.com/products/notion-mcp) (386↑) — AI 에이전트가 Notion을 직접 읽고 쓰는 MCP 커넥터
- [Pixero AI](https://www.producthunt.com/products/pixero-ai-2) (281↑) — URL 하나로 Meta 광고 캠페인 9분 완성
- [Panorama](https://www.producthunt.com/products/panorama) (288↑, 47댓글) — AI가 팀 워크플로우 패턴을 발견해 자동화 추천
- [Computer Use in Claude Code](https://www.producthunt.com/products/claude) (344↑) — Claude가 macOS GUI를 직접 제어

**왜 주목해야 하나**:
W13의 에이전트 "운영 인프라" 트렌드가 W14에서 **실행 에이전트 제품의 폭발**로 이어졌다. Jupid의 "은행 연결 → 세금 자동 분류", Pixero의 "URL → 광고 캠페인 완성", Panorama의 "업무 패턴 자동 발견"은 모두 사용자가 프롬프트를 쓰지 않아도 에이전트가 알아서 실행하는 패턴이다. PH 상위 10개 중 7개가 AI 에이전트/자동화 제품인 날(04-01, 04-03)이 2회 발생.

**1인 개발자 기회**:
"LLM이 못하는 특정 영역"에 데이터 레이어를 추가하는 Jupid 패턴이 가장 재현 가능한 전략. 한국 세무/회계, 부동산 계약서 분석, 식단 영양 계산 등 도메인 특화 데이터 레이어 + AI 에이전트 조합. 또는 Notion MCP처럼 아직 MCP 커넥터가 없는 도구(Airtable, Linear, Google Sheets)용 커넥터 개발.

### 2. 로컬 AI 실행 주류화 — 클라우드 없는 AI 시대 개막 🔥 Rising
**이번 주 등장 횟수**: 6회
**관련 제품/기사**:
- [Ollama v0.19](https://www.producthunt.com/products/ollama) (351↑, 4.96평점) — MLX 기반 Apple Silicon 재구축, 로컬 성능 극적 향상
- [Gemma 4 오픈소스](https://deepmind.google/models/gemma/gemma-4/) (HN 1,126점, Google Trends 500+) — Apache 2.0 라이선스, NVIDIA RTX 최적화
- [AMD Lemonade](https://lemonade-server.ai) (HN 439점) — AMD GPU+NPU 활용 오픈소스 로컬 LLM 서버
- [Tiny Aya 3.35B](https://www.producthunt.com/products/cohere-2) (PH 192↑, 4.92평점) — 70+ 언어, 스마트폰 실행 가능
- Gemma 4 아이폰 실행 (HN 431점) — 모바일에서도 고품질 AI 가능
- Apple M5 Max 로컬 AI (YouTube 36만 뷰) — 통합 메모리로 대형 LLM 로컬 실행

**왜 주목해야 하나**:
한 주에 Google, AMD, Apple, Cohere가 동시에 로컬 AI 실행을 강화한 것은 우연이 아니다. Ollama v0.19의 MLX 리빌드 + Gemma 4 오픈소스 + AMD Lemonade의 조합은 **NVIDIA CUDA 독점을 깨고 로컬 AI의 민주화**를 가속하고 있다. Tiny Aya가 70개 이상 언어를 3.35B 파라미터로 지원하며 "스마트폰에서 실행 가능한 다국어 AI"가 현실화.

**1인 개발자 기회**:
Gemma 4 + Ollama로 API 비용 제로의 AI 제품 구축이 가능해졌다. 프라이버시 민감 도메인(건강 데이터 분석, 법률 문서 검토, 일기 분석)에서 "데이터가 기기를 떠나지 않는" 오프라인 AI 앱이 차별점. Sono(한국인 개발자, $4.99, iOS #37)가 이 전략으로 성공한 실증 사례.

### 3. 바이브코딩 논쟁 — 개발 문화 전쟁으로 확대 ⚡ Inflection Point
**이번 주 등장 횟수**: 5회
**관련 제품/기사**:
- "8년 원했던 것을 AI로 3개월에 만들었다" (HN 626점) — 바이브코딩 성공 사례
- "당신의 코드는 가치가 없다" (Lobsters 45점) — AI 시대 코드 가치 하락론
- "Claude Code가 23년 묵은 리눅스 취약점 발견" (Lobsters 79점) — AI가 인간을 초월하는 코드 감사
- "코드 작성 속도가 문제라면 더 큰 문제가 있다" (Lobsters 52점) — 바이브코딩 비판
- "Slop(AI 저질 코드)이 반드시 미래는 아니다" (HN 187점, 337댓글) — 품질 vs 속도 논쟁

**왜 주목해야 하나**:
W13의 "바이브코딩 품질 논쟁"이 W14에서 **개발자 정체성 논쟁**으로 격화되었다. 한쪽에서는 "8년 프로젝트를 3개월에 완성"(626점)이 올해 Show HN 최고 수준의 반응을 얻었고, 다른 쪽에서는 "코드 자체의 가치가 떨어지고 있다"는 존재론적 위기감이 표출되었다. Claude Code가 23년 묵은 리눅스 취약점을 발견한 사건은 "AI가 코드 작성뿐 아니라 코드 감사에서도 인간을 넘어선다"는 증거로 양측 모두에게 인용되고 있다.

**1인 개발자 기회**:
바이브코딩 품질 검증 도구(AI 생성 코드 리뷰어), AI 코딩 프로세스 템플릿/가이드 콘텐츠, 또는 AI 생성 코드 전용 보안 스캐너. "AI로 만들되, 품질은 AI로 검증한다"는 파이프라인에 수요가 급증 중.

- [Claude Code Voice Mode](https://www.producthunt.com/products/claude-code-voice-mode) (283↑) — 코딩 에이전트에 음성 인터페이스 추가
- [Mercury Edit 2](https://www.producthunt.com/products/mercury-412) (148↑) — "다음 편집 예측" 디퓨전 LLM

**왜 주목해야 하나**:
Google이 Vids 2.0을 **무료**로 내놓으며 Canva Video, Lumen5 같은 유료 영상 도구 시장을 직접 공격했다. Veo 3.1 Lite의 50% 비용 절감은 대량 영상 생산 시대를 예고한다. AI 코딩 도구도 "코드 완성"에서 Mercury Edit 2의 "편집 예측", Voice Mode의 "음성 코딩", Handle Extension의 "비주얼 디버깅"으로 다방면 분화가 가속.

**1인 개발자 기회**:
빅테크가 무료로 진입한 범용 영상 시장 대신 버티컬 영상 템플릿(부동산, 식당 메뉴, SaaS 데모)이 기회. AI 코딩 도구의 "보조 도구" 시장도 열리고 있다 — Handle Extension(브라우저에서 UI 수정→코딩 에이전트 전달, 134↑, 5.0평점)이 좋은 사례.

- LinkedIn 브라우저 확장 스캔 폭로 (HN 1,552점, 681댓글) — 역대급 프라이버시 반응

- WhatsApp 가짜 앱 = 정부 스파이웨어 (TechCrunch) — 모바일 보안 충격

**왜 주목해야 하나**:
W13의 PyPI 공급망 공격(LiteLLM/Telnyx)에 이어 W14에서 **Axios npm 공격 + LinkedIn 스캐닝 + Claude Code 유출**이 동시에 터지며 보안/프라이버시 위기가 "구조적 문제"임이 재확인되었다. LinkedIn 기사가 HN 1,552점을 기록한 것은 개발자 커뮤니티의 프라이버시 감수성이 극에 달했음을 보여준다. Cloudflare가 EmDash로 WordPress 보안 문제를 정면 공략한 것(520점)도 같은 맥락.

**1인 개발자 기회**:
브라우저 확장 권한 감사 도구, npm/PyPI 의존성 보안 스캐너(`npx ai-sbom`), 프라이버시 친화 분석 도구(Sleek Analytics가 PH 2위 237↑를 기록하며 수요 확인). WordPress→보안 CMS 마이그레이션 도구도 기회.
