### 1. Kilo Code Reviewer ⬆️ 568
![thumbnail](https://ph-files.imgix.net/1a41e8bf-2f29-42ee-8bce-d99357ebc0fb.gif?auto=format)

#### 이 제품은 무엇인가?
PR(풀 리퀘스트)을 열면 AI가 자동으로 코드 리뷰를 해주는 도구다. 기존에는 팀원에게 리뷰를 요청하고 기다려야 했지만, 이 도구를 쓰면 PR을 열자마자 즉시 AI가 코드 품질, 버그 가능성, 개선점을 분석해서 코멘트를 남긴다.

GitHub Actions와 연동되어 설정 한 번이면 모든 PR에 자동 적용된다. 오픈소스로 공개되어 있어 무료로 사용 가능하다.

#### 왜 주목할만한가?
1인 개발자는 코드 리뷰를 받을 사람이 없다. 이 도구로 "혼자서도 리뷰받는 환경"을 만들 수 있다. 568표를 받으며 오늘 1위를 차지한 것은 이 필요가 그만큼 크다는 증거다.

| 구분 | 내용 |
|------|------|
| 타겟 사용자 | 코드 리뷰 인력이 부족한 개발팀, 1인 개발자 |
| 기존 대비 차별점 | PR 열자마자 즉시 리뷰 (기존 도구는 수동 요청 필요) |
| 수익 모델 | 오픈소스 무료 (GitHub 스폰서십 가능) |
| 제작자 | 7명 팀 |

**배울 점**: 오픈소스 + 실용적인 문제 해결 = 빠른 입소문. 개발자 도구는 직접 써보고 좋으면 퍼진다.

### 2. Moltbot ⬆️ 478
![thumbnail](https://ph-files.imgix.net/a15ae988-f9b3-403f-b470-a5ef1d15162c.png?auto=format)

#### 이 제품은 무엇인가?
"실제로 행동하는 AI 비서"를 표방하는 로컬 AI 에이전트다. 기존 챗봇은 "이렇게 하세요"라고 설명만 해주지만, Moltbot은 파일을 직접 수정하고, 명령어를 실행하고, 시스템 설정을 바꾸는 등 실제 작업을 수행한다.

로컬에서 돌아가기 때문에 클라우드에 데이터를 보내지 않아도 된다. 오픈소스로 공개되어 있다.

#### 왜 주목할만한가?
"AI가 말만 하지 말고 직접 해줬으면"이라는 불만이 커지고 있다. Moltbot은 이 니즈를 정면으로 공략한다. 1인 제작자가 만들어서 478표를 받은 것도 인상적이다.

| 구분 | 내용 |
|------|------|
| 타겟 사용자 | AI 챗봇의 "설명만 하는" 방식에 지친 파워유저 |
| 기존 대비 차별점 | 설명이 아닌 실행 - 파일 수정, 명령어 실행 등 직접 수행 |
| 수익 모델 | 오픈소스 무료 |
| 제작자 | 1인 솔로 개발자 |

**배울 점**: "실제로 동작하는 것"이 차별화. 기능이 적어도 확실히 작동하면 주목받는다.

### 3. LobeHub ⬆️ 338
![thumbnail](https://ph-files.imgix.net/d91fc33b-29d2-4222-b0dc-977c6a78a670.png?auto=format)

#### 이 제품은 무엇인가?
AI 에이전트를 "도구"가 아닌 "팀원"으로 바라보는 플랫폼이다. 사용자가 자신만의 AI 에이전트를 만들고, 이 에이전트들이 함께 성장하며 협업하는 구조다.

마케팅 자동화, 생산성 향상 등 다양한 용도의 에이전트 템플릿을 제공한다. GitHub에서 활발히 개발 중인 오픈소스 프로젝트다.

#### 왜 주목할만한가?
"AI = 도구"에서 "AI = 동료"로 인식이 바뀌고 있다. 1인 개발자 입장에서 AI 에이전트를 "나와 함께 성장하는 팀원"으로 포지셔닝하면 차별화가 된다는 것을 보여준다.

| 구분 | 내용 |
|------|------|
| 타겟 사용자 | AI 에이전트를 적극 활용하려는 스타트업, 1인 창업자 |
| 기존 대비 차별점 | 에이전트 = 팀원이라는 컨셉, 함께 성장하는 구조 |
| 수익 모델 | 오픈소스 (프리미엄 기능 예상) |
| 제작자 | 5명 팀 |

**배울 점**: 기술보다 "프레이밍"이 중요할 때가 있다. 같은 AI 에이전트도 "도구" vs "팀원"으로 포지셔닝하면 느낌이 완전히 다르다.

### 4. CapslockMute ⬆️ 160
![thumbnail](https://ph-files.imgix.net/d503d1c4-1938-456b-99d2-310423276aea.png?auto=format)

#### 이 제품은 무엇인가?

회의 중에 기침하거나, 배경 소음이 들릴 때 앱을 전환하지 않고 바로 음소거할 수 있다.

#### 왜 주목할만한가?
단 하나의 기능만 있는 앱이 160표를 받았다. 리모트 워커들의 일상적인 불편함을 정확히 짚었기 때문이다. 1인 개발자가 만들었다.

| 구분 | 내용 |
|------|------|
| 타겟 사용자 | 화상회의가 많은 리모트 워커 |
| 기존 대비 차별점 | 앱 전환 없이 단축키 하나로 모든 앱 음소거 |
| 수익 모델 | 맥 앱 (유료 가능성 높음) |
| 제작자 | 1인 솔로 개발자 |

**배울 점**: 단일 기능 유틸리티가 여전히 통한다. "작지만 확실한 불편함"을 해결하면 된다.
