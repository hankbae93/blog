# PRD: LocalFlash

**로컬-퍼스트 PDF to Flashcard 학습 앱**

---

## 1. 개요

### 1.1 문제 정의

의대생, 법대생 등 전문직 수험생들은 **민감한 학습 자료**를 다룬다:
- 의대생: 환자 케이스 스터디, 의료 기록
- 법대생: 기밀 판례, 클라이언트 정보
- 기업 연수생: 사내 문서, 영업 비밀

이들은 효과적인 암기를 위해 Anki, Quizlet 같은 플래시카드 앱을 쓰고 싶지만:

**핵심 문제:**
- **프라이버시 우려**: "환자 정보를 클라우드에 올릴 수 없다" (HIPAA, 개인정보보호법)
- **수동 작업 부담**: "PDF에서 플래시카드 만드는 데 시간이 너무 걸린다"
- **AI-프라이버시 트레이드오프**: "AI 자동 생성은 좋지만 데이터 유출이 걱정"

**방치된 이유:**
- 기존 도구(Anki, Quizlet)는 클라우드 동기화 기반
- 로컬 LLM이 최근에야 실용적 수준 도달
- 틈새 시장이라 대기업이 관심 없음

### 1.2 솔루션

**LocalFlash**는 **100% 로컬**에서 동작하는 PDF to Flashcard 앱:
1. PDF를 **기기에서 직접 파싱** (업로드 없음)
2. **로컬 LLM**으로 플래시카드 자동 생성
3. **간격 반복(Spaced Repetition)**으로 효과적 학습
4. 데이터가 **기기 밖으로 절대 나가지 않음**

### 1.3 성공 지표 (MVP 기준)

| 지표 | 목표 | 측정 방법 |
|------|------|----------|
| 다운로드 | 1,000건 | App Store/직접 배포 |
| 일간 활성 | 200명 | 일 1회 이상 학습 |
| 플래시카드 생성 정확도 | 85%+ | 사용자 피드백 |
| 유료 전환율 | 10%+ | Free → Pro |

## 2. 타겟 사용자

### 2.1 Primary Persona: 김의대 (26세, 의대 본과 3학년)

**배경:**
- 서울 소재 의대 재학
- MacBook Pro M3 사용
- Anki로 공부하다가 프라이버시 걱정으로 중단

**Pain Points:**
- "교수님이 주신 환자 케이스를 Anki에 넣으려다가... 이거 환자 정보인데 괜찮나?"
- "결국 수동으로 카드 만드는데 한 챕터에 2시간씩 걸림"
- "ChatGPT로 만들면 빠른데, 환자 정보를 OpenAI에 보내는 게 찝찝"

**행동 패턴:**
- 새벽까지 공부, 효율 중시
- 개인정보 민감 (의료윤리 교육 받음)
- Apple 생태계 선호

### 2.2 Secondary Persona: 박변시 (29세, 로스쿨 2학년)

**배경:**
- 변호사시험 준비
- 사무실 인턴 중, 실제 사건 자료 접근
- 판례 암기가 핵심

**Pain Points:**
- "사건 자료로 공부하고 싶은데 기밀이라..."
- "판례 요약을 일일이 타이핑하기 귀찮음"
- "기존 앱들은 법률 용어를 잘 못 알아듣는 것 같음"

### 2.3 사용 시나리오

**시나리오 1: 김의대의 케이스 스터디**
1. 교수님이 PDF로 환자 케이스 10건 배포
2. LocalFlash에서 PDF 열기 (업로드 없이 로컬에서)
3. "플래시카드 생성" 버튼 클릭
4. 로컬 LLM이 진단명, 증상, 치료법 추출 → 카드 생성
5. 생성된 카드 검토 및 편집
6. 간격 반복으로 매일 학습

**시나리오 2: 박변시의 판례 암기**
1. 법원 판결문 PDF 다운로드
2. LocalFlash로 불러오기
3. "법률 용어 모드"로 카드 생성
4. 쟁점, 판결 요지, 적용 법조문 카드화
5. 통학 시간에 복습

## 3. 기능 요구사항

### 3.1 MVP 기능 (v1.0)

#### F1. PDF 로컬 파싱
**설명:** PDF를 기기에서 직접 텍스트 추출
- 네트워크 요청 없이 완전 오프라인 처리
- 이미지 PDF는 로컬 OCR 지원
- 한글/영어 지원

**수용 기준:**
- [ ] 100페이지 PDF를 60초 내 파싱
- [ ] 네트워크 모니터로 외부 요청 0건 확인
- [ ] 한글 PDF 정확도 95%+

#### F2. 로컬 LLM 플래시카드 생성
**설명:** 로컬 LLM으로 텍스트에서 Q&A 카드 자동 생성
- llama.cpp + MLX 기반
- 커스텀 프롬프트 지원
- 도메인 모드 (의학/법학/일반)

**수용 기준:**
- [ ] 1페이지당 3-5개 카드 생성
- [ ] 생성 시간 페이지당 30초 이내 (M1 기준)
- [ ] 사용자 피드백 "유용함" 85%+

#### F3. 간격 반복 학습
**설명:** SM-2 알고리즘 기반 복습 스케줄
- 난이도별 다음 복습일 자동 계산
- 오늘의 학습량 표시
- 학습 통계 대시보드

**수용 기준:**
- [ ] SM-2 알고리즘 정확 구현
- [ ] 복습 알림 (로컬 알림)
- [ ] 주간/월간 학습 통계

### 3.2 향후 기능 (v2.0+)

| 우선순위 | 기능 | 설명 |
|---------|------|------|
| High | iOS/iPadOS 버전 | Apple Silicon iPad 지원 |
| High | 이미지 카드 | 의학 이미지 포함 카드 |
| Medium | Anki 내보내기 | .apkg 파일 생성 |

### 3.3 명시적 제외 범위

- **클라우드 동기화**: 절대 구현 안 함 (핵심 가치)
- **웹 버전**: 로컬 LLM 불가
- **Windows 버전**: MVP에서 제외 (추후 검토)
- **소셜 기능**: 학습 공유, 랭킹 등

```
│  │  │ Mistral 7B  │  │ Llama 3 8B  │      │        │

│           │                                         │
│  ┌────────▼────────────────────────────────┐       │
│  │           Local Database                 │       │
│  │             (SQLite)                     │       │
│  │   Cards / Decks / Study History          │       │
│  └──────────────────────────────────────────┘       │
│                                                      │
│  ┌──────────────────────────────────────────┐       │
│  │        Spaced Repetition Engine          │       │
│  │              (SM-2)                      │       │
│  └──────────────────────────────────────────┘       │
└─────────────────────────────────────────────────────┘

      ❌ NO NETWORK CALLS ❌
```

### 4.2 기술 스택

| 영역 | 기술 | 선택 이유 |
|------|------|----------|
| App Framework | Tauri 2.0 | 작은 번들, Rust 성능, macOS 네이티브 |
| Frontend | React + TailwindCSS | 빠른 개발 |
| Local LLM | llama.cpp + MLX | Apple Silicon 최적화 |
| Default Model | Mistral 7B Q4 | 품질/크기 밸런스 |
| PDF Parser | pdf-extract (Rust) | 빠른 파싱, 오프라인 |
| OCR | Apple Vision Framework | 네이티브, 정확도 높음 |
| Database | SQLite | 가볍고 안정적 |
| SR Algorithm | SM-2 | 검증된 알고리즘 |

### 4.3 성능 요구사항

| 항목 | 요구사항 | 측정 환경 |
|------|----------|----------|
| PDF 파싱 | < 1초/페이지 | 100페이지 PDF |
| 카드 생성 | < 30초/페이지 | M1 MacBook Air |
| 앱 시작 | < 3초 | Cold start |
| 모델 로드 | < 10초 | 첫 사용 시 |
| 앱 크기 | < 5GB | 모델 포함 |

### 4.4 데이터 모델

```sql
-- 덱 (카드 묶음)
decks (
  id INTEGER PRIMARY KEY,
  name TEXT,
  created_at TIMESTAMP,
  source_pdf_path TEXT
)

-- 플래시카드
cards (
  id INTEGER PRIMARY KEY,
  deck_id INTEGER REFERENCES decks,
  front TEXT,          -- 질문
  back TEXT,           -- 답
  source_page INTEGER, -- 원본 페이지
  created_at TIMESTAMP
)

-- 학습 기록
study_logs (
  id INTEGER PRIMARY KEY,
  card_id INTEGER REFERENCES cards,
  quality INTEGER,     -- 0-5 (SM-2)
  interval INTEGER,    -- 다음 복습까지 일수
  easiness REAL,       -- E-Factor
  reviewed_at TIMESTAMP
)
```

## 8. 부록

### 8.1 품질 검증 계획

**로컬 LLM vs GPT-4 비교:**

| 항목 | 측정 방법 | 목표 |
|------|----------|------|
| 카드 정확도 | 수동 검토 100장 | GPT-4 대비 80% |
| 생성 속도 | 10페이지 PDF | 5분 이내 |
| 한글 품질 | 의학 용어 포함 | 90% 정확 |

### 8.2 페르소나 검증 결과

**전제 파괴:** ✅
- "AI 기능 쓰려면 클라우드에 데이터 올려야 한다" → 로컬에서 가능

**방치된 이유:** ✅
- 로컬 LLM이 최근에야 실용적. Apple Silicon + MLX 조합이 2024년부터 가능

**비합리적 행동:** ✅
- 의대생이 환자 케이스를 Anki에 넣으려다 포기, 수동으로 카드 만듦

**기능-가설 연결:** ✅
- PDF 로컬 파싱(업로드 불필요) + 자동생성(노동 제거) + 간격반복(효과적 암기)

**진정성 검증 필요:**
- "로컬 LLM의 품질이 GPT-4 대비 얼마나 떨어지는가?"
- → 대응: MVP 출시 전 비교 테스트 필수, 사용자가 품질-프라이버시 트레이드오프 수용하는지 검증

### 8.3 경쟁사 분석
