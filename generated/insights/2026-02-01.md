### 2. skills.sh ⬆️ 261
![thumbnail](https://ph-files.imgix.net/593ff5e2-9d11-491c-ab63-9e929e894214.jpeg?auto=format)

#### 이 제품은 무엇인가?
AI 에이전트를 위한 스킬 디렉토리다. 에이전트에 특정 기능(스킬)을 추가하고 싶을 때, `npx skills add`라는 한 줄 명령어로 설치할 수 있다. 스킬은 에이전트가 절차적 지식에 접근할 수 있게 해주는 재사용 가능한 컴포넌트다.

#### 왜 주목할만한가?
npm이 JavaScript 생태계를 만들었듯, skills.sh는 AI 에이전트 생태계의 "패키지 매니저"가 되려 한다. 에이전트가 보편화되면 "어떤 스킬을 조합할 것인가"가 제품 차별화 요소가 된다. 이 플랫폼 위치를 선점하면 엄청난 네트워크 효과를 얻을 수 있다.

| 구분 | 내용 |
|------|------|
| 타겟 사용자 | AI 에이전트 개발자, Claude Code/Cursor 사용자 |
| 기존 대비 차별점 | 원클릭 스킬 설치, 절차적 지식 공유 플랫폼 |
| 수익 모델 | 미정 (플랫폼/마켓플레이스 잠재력) |
| 제작자 | 팀 |

**배울 점**: 새로운 기술 생태계가 형성될 때, "디렉토리/마켓플레이스"는 항상 기회다. 에이전트 스킬 생태계는 지금 초기 단계.

### 3. Pretty Prompt 1.0 ⬆️ 250
![thumbnail](https://ph-files.imgix.net/77d7fdfe-35a2-4e62-8353-e02e185613f7.jpeg?auto=format)

#### 이 제품은 무엇인가?
"프롬프트를 위한 Grammarly"다. 브라우저 확장 프로그램으로, ChatGPT, Claude, Gemini, Perplexity 등 어떤 AI 도구에서든 텍스트를 선택하면 원클릭으로 더 좋은 프롬프트로 변환해준다. 브라우저를 벗어나지 않고도 프롬프트를 개선할 수 있다.

#### 왜 주목할만한가?
"X를 위한 Grammarly"는 검증된 포지셔닝이다. 프롬프트 작성은 모든 AI 사용자의 공통 과제인데, 대부분 "어떻게 더 잘 쓸 수 있을까"를 모른다. 이 불편함을 브라우저 확장이라는 마찰 없는 형태로 해결했다.

| 구분 | 내용 |
|------|------|
| 타겟 사용자 | AI 도구를 자주 쓰지만 프롬프트 작성에 익숙하지 않은 사용자 |
| 기존 대비 차별점 | 브라우저에서 바로 프롬프트 개선, 모든 AI 도구 지원 |
| 수익 모델 | 프리미엄 |
| 제작자 | 2명 팀 |

**배울 점**: 기존 성공 제품(Grammarly)의 모델을 새 영역(프롬프트)에 적용하는 전략은 여전히 유효하다.

### 4. Herodot AI ⬆️ 181
![thumbnail](https://ph-files.imgix.net/8edd81ce-4b29-4198-beb2-e6ae02f29995.png?auto=format)

#### 이 제품은 무엇인가?

#### 왜 주목할만한가?
여행 가이드 앱은 많지만, "현지인 친구처럼 이야기해주는" 경험을 AI로 구현한 건 차별화 포인트다. 2년이라는 개발 기간은 길지만, 그만큼 품질에 투자했다는 의미이기도 하다.

| 구분 | 내용 |
|------|------|
| 타겟 사용자 | 여행객, 역사/문화에 관심 있는 사용자 |
| 기존 대비 차별점 | AI가 "현지인 친구"처럼 자연스러운 스토리텔링 |
| 수익 모델 | 프리미엄 (기본 무료) |
| 제작자 | 2명 팀 |

**배울 점**: AI로 "경험"을 제공하는 제품은 단순 정보 제공보다 강력하다. "로컬 가이드 친구"라는 포지셔닝이 감정적 연결을 만든다.

### 5. RUSTWA ⬆️ 39
![thumbnail](https://ph-files.imgix.net/4ce7bc9d-aa1c-4da9-ab15-acdded523eab.png?auto=format)

#### 이 제품은 무엇인가?
Rust로 만든 비공식 WhatsApp Web API다. 텍스트/미디어 전송, 웹훅 메시지 수신, 멀티 세션 지원, QR 로그인, 세션 유지, 웹 대시보드까지 "90% 유스케이스"를 커버한다. Tokio 기반 비동기 런타임으로 빠르다.

#### 왜 주목할만한가?
기존 Node.js 기반 WhatsApp 라이브러리(whatsapp-web.js)의 Rust 버전이다. "기능 과다 없이 핵심만"이라는 철학이 명확하고, Rust의 성능/안정성 장점을 살렸다. 오픈소스지만 WhatsApp 자동화 수요가 높아 후속 수익화 기회가 있다.

| 구분 | 내용 |
|------|------|
| 타겟 사용자 | WhatsApp 봇/자동화 개발자 |
| 기존 대비 차별점 | Rust + Tokio로 성능 향상, 핵심 기능만 집중 |
| 수익 모델 | 오픈소스 |
| 제작자 | 솔로 |

**배울 점**: 기존 인기 라이브러리를 더 빠른 언어로 재구현하는 건 틈새 기회다. Rust 재작성 트렌드가 계속되고 있다.

- **경고**: 범용 에이전트는 빅테크와 경쟁해야 한다. 니치 영역에 집중하라.
