# /generate - PRD 문서 생성 커맨드

인사이트 분석 결과를 바탕으로 상세 PRD(Product Requirements Document) 마크다운 파일을 생성합니다.

## 트랙 옵션

| 커맨드 | 설명 |
|--------|------|
| `/generate` | 투 트랙 실행 (essence + profit) |
| `/generate:essence` | 🧠 본질 추구형만 |
| `/generate:profit` | 💰 수익화 특화만 |

---

## 입력
- 본질 트랙: `generated/insights/{YYYY-MM-DD}-essence.md`
- 수익 트랙: `generated/insights/{YYYY-MM-DD}-profit.md`
- 기본: 각 트랙 상위 3개 아이디어에 대해 PRD 생성
- 옵션: 특정 아이디어만 지정 가능 (예: `/generate MCPmarket`)

## 출력
- 본질 트랙: `generated/generated/output/prd-{아이디어명}-essence-{날짜}.md`
- 수익 트랙: `generated/generated/output/prd-{아이디어명}-profit-{날짜}.md`

---

## 생성 지침

### 1. 인사이트 파일 로드

**트랙별 입력 파일:**
- `/generate` (투 트랙): 두 파일 모두 로드
- `/generate:essence`: `generated/insights/{날짜}-essence.md`
- `/generate:profit`: `generated/insights/{날짜}-profit.md`

```
각 인사이트 파일에서 top_ideas 섹션을 파싱합니다.
```

### 2. 아이디어별 PRD 생성

각 아이디어에 대해:
1. 인사이트의 요약 정보를 기반으로 PRD 템플릿 채우기
2. 창의적으로 세부 사항 확장 (사용자 시나리오, 기술 스택 등)
3. 현실적인 마일스톤 및 지표 설정

**트랙별 강조점:**

#### 🧠 Essence Track PRD
- **문제 정의** 섹션을 특히 깊게 작성
- "전제 파괴", "방치된 이유" 관점 명시
- 본질적 가치에 집중한 MVP 기능

#### 💰 Profit Track PRD
- **수익 모델** 섹션을 특히 상세하게 작성
- 구체적 가격 ($XX/월) 명시
- 첫 수익까지의 타임라인 명시
- CAC vs LTV 추정치 포함

### 3. 품질 체크리스트

**공통:**
- [ ] 문제 정의가 구체적이고 측정 가능한가?
- [ ] 타겟 사용자가 "개발자" 같은 모호한 정의가 아닌가?
- [ ] MVP 기능이 3개 이하인가?
- [ ] 기술 스택 선택에 이유가 있는가?
- [ ] 마일스톤이 2주 내 달성 가능한가?

**🧠 Essence Track 추가 체크:**
- [ ] 전제 파괴 관점이 명확한가?
- [ ] 문제의 본질을 건드리고 있는가?

**💰 Profit Track 추가 체크:**
- [ ] 수익 모델이 구체적인가? (가격 명시)
- [ ] 30일 내 첫 수익 가능한가?
- [ ] 반복 매출 구조인가?

### 4. 파일 저장

**트랙별 파일명 규칙:**
```
본질 트랙: generated/generated/output/prd-{아이디어명 소문자}-essence-{YYYY-MM-DD}.md
수익 트랙: generated/generated/output/prd-{아이디어명 소문자}-profit-{YYYY-MM-DD}.md
```

예:
- `generated/generated/output/prd-mcpmarket-essence-2026-01-26.md`
- `generated/generated/output/prd-reviewbot-profit-2026-01-26.md`
