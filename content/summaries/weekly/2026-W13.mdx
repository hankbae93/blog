## 1. 주간 트렌드 TOP 5

### 1. AI 에이전트 성숙 사이클: 생성→관찰→수익화→오케스트레이션 🔥 Rising
**이번 주 등장 횟수**: 7회 (매일)
**관련 제품/기사**:
- [Agentplace AI Agents](https://www.producthunt.com/posts/agentplace-ai-agents) (496↑) — 에이전트 빌더 플랫폼
- [Crossnode](https://www.producthunt.com/posts/crossnode) (286↑, 57댓글) — 에이전트→유료 SaaS 변환기
- [Cekura](https://www.producthunt.com/posts/cekura) (361↑) — AI 에이전트 관찰성 플랫폼
- [Parallel Code](https://www.producthunt.com/posts/parallel-code) (146↑) — 멀티에이전트 git worktree 관리
- [Cline Kanban](https://www.producthunt.com/posts/cline-kanban) (144↑) — 에이전트 태스크 칸반보드
- [CrabTalk](https://www.producthunt.com/posts/crabtalk) (170↑) — 8MB 경량 에이전트 데몬
- [Agentation](https://www.producthunt.com/posts/agentation) (352↑) — UI 어노테이션으로 에이전트 학습

**왜 주목해야 하나**:
W12까지 에이전트의 "수직 확장"(코딩→디자인→CRM)이 핵심이었다면, W13은 에이전트의 **운영 인프라**가 폭발했다. Crossnode의 "에이전트→SaaS 변환"은 수익화를, Cekura는 관찰성을, Parallel Code/Cline Kanban은 오케스트레이션을 해결한다. PH 상위 10개 중 7개가 AI/에이전트 관련이 된 날(03-29)이 이 흐름의 정점.

**1인 개발자 기회**:
에이전트 수익화 대시보드 — Crossnode처럼 에이전트를 SaaS로 패키징하되, 결제·사용량·고객 관리를 한 화면에서 처리하는 "Stripe for Agents" 미들웨어. 또는 Cline Kanban 방식의 에이전트 태스크 매니저를 VS Code 외 에디터(Neovim, JetBrains)로 확장.

### 2. AI 공급망 보안 위기 — PyPI 연쇄 공격 🚨 Critical
**이번 주 등장 횟수**: 4회
**관련 제품/기사**:
- LiteLLM PyPI 악성코드 사건 (HN 484pts, 03-25)
- LiteLLM 실시간 대응 후속 (HN 290pts, 03-27)
- Telnyx PyPI 공급망 공격 (03-28, 연속 2일째)
- Claude Code `.claude/` 폴더 구조 분석 (HN 347pts) — 에이전트 보안 아키텍처 관심

**왜 주목해야 하나**:
같은 주에 LiteLLM과 Telnyx 두 개의 PyPI 공급망 공격이 연쇄 발생한 것은 AI 도구 생태계의 보안 취약점이 구조적 문제임을 보여준다. 에이전트가 `pip install`로 의존성을 자동 설치하는 "바이브코딩" 환경에서 공급망 공격의 파급력이 기존보다 훨씬 크다. Claude Code 아키텍처에 대한 HN의 높은 관심(347pts)도 "에이전트가 내 시스템에서 뭘 하는가"에 대한 불안의 반영.

**1인 개발자 기회**:
AI 프로젝트 전용 SBOM(소프트웨어 부품 목록) 생성기 — `npx ai-sbom` 한 줄로 프로젝트의 모든 AI 관련 의존성을 스캔, 알려진 취약점 매칭, 리스크 스코어 출력. 또는 에이전트 샌드박싱 도구(에이전트의 파일시스템·네트워크 접근을 컨테이너로 격리).

**이번 주 등장 횟수**: 5회
**관련 제품/기사**:
- [Apparent for Gmail](https://www.producthunt.com/posts/apparent-for-gmail) (172↑) — Gmail에서 AI 기능 제거 확장
- GitHub→Codeberg 이주 급증 (HN 515pts, 03-27)

- AI 챗봇 과잉 사과 연구 (HN 498pts, 03-29)
- 바이브코딩 품질 논쟁 "90% Claude 출력이 별점 2 미만" (HN, 03-26)

**왜 주목해야 하나**:
W12의 "EnshittifAIcation" 담론이 W13에서 **실제 제품과 행동**으로 전환되었다. Apparent for Gmail은 "AI를 빼는 것"이 제품이 된 첫 사례이고, GitHub→Codeberg 이주(515pts)는 플랫폼 종속에 대한 개발자 행동이 가시화된 것이다. 동시에 AI 챗봇의 과잉 사과 연구(498pts)는 현재 AI UX의 한계에 대한 학술적 검증.

**1인 개발자 기회**:
"De-tech" 브라우저 확장 시리즈 — Gmail뿐 아니라 YouTube, LinkedIn, Twitter에서 AI 추천/AI 생성 콘텐츠를 필터링하는 확장 모음. "AI-free 모드"를 토글로 제공. 또는 GitHub→Forgejo 원클릭 마이그레이션 CLI + Forgejo 플러그인 마켓플레이스.
