# /weekly-ideas - 주간 아이디어 브리프

이번 주의 데일리 인사이트와 소스 데이터를 종합하여 **1인 개발자를 위한 주간 아이디어 브리프**를 생성합니다.

## 입력

- `generated/insights/YYYY-MM-DD.md` (이번 주 모든 파일 - 아이디어 카드 섹션 참조)
- `generated/sources/YYYY-MM-DD.json` (이번 주 모든 소스 - 특히 `app_reviews` 데이터)
- `generated/app-trends/` (앱 트렌드 히스토리)

## 실행 조건

매주 일요일에 실행하며, 지난 7일간의 데이터를 종합합니다.

## 핵심 목표

**"이거 해볼까?" 순간을 만들어내는 것.**

데일리 인사이트의 아이디어 카드가 "힌트"라면, 위클리 아이디어 브리프는 "실행 가이드"입니다.
추상적인 기회 제시가 아니라, 구체적인 제품 스펙과 시장 검증 데이터를 제공합니다.

## 분석 프로세스

### Step 1: 주간 데이터 수집
1. 이번 주 7일간의 `generated/sources/YYYY-MM-DD.json` 파일을 모두 읽기
2. 이번 주 7일간의 `generated/insights/YYYY-MM-DD.md` 파일에서 "아이디어 카드" 섹션 추출
3. `app_reviews` 소스에서 반복 등장하는 불만 테마 집계

### Step 2: 신호 강도 분석
- **반복 등장 테마**: 같은 불만/기회가 3일+ 등장하면 강한 신호
- **교차 소스 확인**: PH + 앱 리뷰 + HN에서 같은 키워드 → 검증된 시장
- **앱 리뷰 불만 집계**: 동일 테마 불만이 여러 앱에서 발견 → 카테고리 전체 문제

### Step 3: 아이디어 심층 분석 (5-10개)
- 데일리 아이디어 카드 중 가장 유망한 것들을 선별하여 심층 분석
- 새로운 조합도 생성 (주간 데이터를 가로질러 조합)
- 각 아이디어에 대해 아래 출력 구조대로 상세 분석

## 파일 저장

- **경로**: `generated/ideas/weekly/{YYYY}-W{WW}.md`
- **인코딩**: UTF-8
- 디렉토리가 없으면 생성

## 주차 계산

ISO 8601 기준으로 주차를 계산합니다.
- 예: 2026-03-09이 속한 주 = 2026-W11

## 분석 지침 (필수 준수)

1. **아이디어 수**: 최소 5개, 최대 10개. 양보다 질 우선
2. **조합 기법 강화**: 주간 데이터를 가로질러 조합 (월요일 PH 제품 + 금요일 앱 리뷰 불만)
3. **반복 신호 우선**: 같은 테마가 여러 날 등장한 아이디어에 높은 우선순위
4. **데이터 기반 검증**: 모든 수익 추정은 수집된 앱 데이터(리뷰수, 순위, IAP)에서 도출
5. **추상적 아이디어 절대 금지**: 모든 아이디어에 조합 공식 + MVP 스펙 + 벤치마크 필수

### 명시적 제외 (이런 아이디어는 생성하지 않음)
- ML 모델 직접 트레이닝 필요
- 하드웨어/IoT 필요
- 규제 산업 (의료기기, 금융 라이선스, 법률 자문)
- 네트워크 효과 1000명+ 없이는 무의미한 서비스
- React/Next.js + Node.js로 구현 불가능한 서비스

### 실현 가능성 평가 기준

| 항목 | 1점 (어려움) | 3점 (보통) | 5점 (쉬움) |
|------|-------------|-----------|-----------|

| 시장 진입 장벽 | 빅테크 독점 | 경쟁자 있지만 불만 존재 | 명확한 공백 |

| 수익화 가능성 | 시장 가격 불명 | 유사 앱 IAP 존재 | 높은 IAP + 많은 리뷰 |
