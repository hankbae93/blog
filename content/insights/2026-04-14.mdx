## 💡 아이디어 카드

### 카드 1: DepWatch — 의존성 소유권 변경 알림 서비스

**조합 공식**: 워드프레스 플러그인 백도어 사건(HN 850점)의 공급망 보안 수요 + GitHub Actions 생태계의 자동화 인프라

**문제 발견 경로**:
- HN: 워드프레스 플러그인 30개 일괄 백도어 — 850점, 238댓글로 업계 최대 관심사
- TechCrunch: Booking.com 해킹 확인 — 보안 사건 연이어 발생
- Lobsters: "누구도 안전한 웹사이트를 강요할 수 없다" — 보안 책임 논쟁 활발

**문제**: 개발자가 사용 중인 npm 패키지, WordPress 플러그인, Chrome 확장의 소유자가 바뀌어도 알 방법이 없다. 신뢰했던 패키지가 인수 후 악성 코드로 변질되는 것을 사전에 감지하지 못함.

**솔루션**: 프로젝트의 의존성 목록을 스캔해, 패키지 maintainer 변경/소유권 이전/비정상 업데이트를 감지하고 즉시 알림을 보내는 서비스.

**MVP 스펙 (핵심 기능 3개)**:
1. package.json/composer.json 파싱 + npm/packagist API로 maintainer 변경 감지 — 핵심 가치
2. GitHub Action으로 CI에 통합 — 기존 워크플로우 침투
3. Slack/이메일 알림 + 변경 상세 리포트 — 즉각 대응 가능

**기술 스택**: Next.js + Vercel Cron + npm Registry API

**수익 검증 데이터**:
